# i18n Design: English + Ukrainian (next-intl, URL prefix)

## Summary

Add full i18n support to keyo.studio with English (en) and Ukrainian (ua) languages. Default locale auto-detected from `Accept-Language` browser header. Language stored in `NEXT_LOCALE` cookie after manual selection. URL-prefix routing (`/en/...`, `/ua/...`) for SEO.

## Key Decisions

- **Library:** `next-intl` (App Router native, TypeScript-safe, ~12kB)
- **Routing:** URL prefix — `/en/` and `/ua/` for all pages
- **Default locale:** `en`
- **Auto-detect:** Yes — from `Accept-Language` HTTP header (`uk` → `/ua/`, else `/en/`)
- **Manual override:** Cookie `NEXT_LOCALE` persists choice across sessions
- **Switcher UI:** `EN | UA` pill in Navbar (between Pricing and user avatar)
- **Scope:** All UI strings including Privacy and Terms pages

## Middleware Architecture

`middleware.ts` chains next-intl and Clerk:

```ts
import createMiddleware from 'next-intl/middleware';
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server';

const intlMiddleware = createMiddleware({
  locales: ['en', 'ua'],
  defaultLocale: 'en',
  localeDetection: true,
});

const isPublicRoute = createRouteMatcher([
  '/:locale/sign-in(.*)',
  '/:locale/sign-up(.*)',
  '/:locale/sso-callback(.*)',
  '/:locale',
  '/:locale/image',
  '/:locale/video',
  '/:locale/audio',
  '/:locale/pricing',
  '/:locale/privacy',
  '/:locale/terms',
  '/api/liqpay/webhook(.*)',
  '/api/community-gallery(.*)',
]);

export default clerkMiddleware((auth, request) => {
  const isIntlPath = !request.nextUrl.pathname.startsWith('/api');
  if (isIntlPath) return intlMiddleware(request);
  if (!isPublicRoute(request)) {
    // protect private routes
  }
});
```

## LocaleSwitcher Component

```tsx
// components/layout/LocaleSwitcher.tsx
'use client';
import { useLocale } from 'next-intl';
import { useRouter, usePathname } from 'next/navigation';

export default function LocaleSwitcher() {
  const locale = useLocale();
  const router = useRouter();
  const pathname = usePathname();

  const toggle = () => {
    const next = locale === 'en' ? 'ua' : 'en';
    // Replace /en/... with /ua/... or vice versa
    const newPath = pathname.replace(/^\/(en|ua)/, `/${next}`);
    router.push(newPath);
  };

  return (
    <button onClick={toggle} className="font-dm" style={{
      display: 'inline-flex', alignItems: 'center', gap: '5px',
      fontSize: '12px', fontWeight: 500,
      padding: '5px 10px', borderRadius: '20px',
      background: 'rgba(255,255,255,0.04)',
      border: '0.5px solid rgba(255,255,255,0.1)',
      cursor: 'pointer', color: 'rgba(255,255,255,0.6)',
      transition: 'all 0.15s',
    }}>
      <span style={{ opacity: locale === 'en' ? 1 : 0.4 }}>EN</span>
      <span style={{ opacity: 0.25 }}>|</span>
      <span style={{ opacity: locale === 'ua' ? 1 : 0.4 }}>UA</span>
    </button>
  );
}
```

Position in Navbar: between Pricing pill and user avatar button.

## SEO Considerations

- `hreflang` alternate links in `app/[locale]/layout.tsx` head:
  ```html
  <link rel="alternate" hreflang="en" href="https://keyo.studio/en/..." />
  <link rel="alternate" hreflang="uk" href="https://keyo.studio/ua/..." />
  <link rel="alternate" hreflang="x-default" href="https://keyo.studio/en/..." />
  ```
- `sitemap.ts` updated to include both locale variants
- OpenGraph locale updated per page: `en_US` / `uk_UA`

## Clerk Integration

Clerk's `<ClerkProvider>` stays in root `app/layout.tsx` (above locale routing). ClerkProvider supports `localization` prop — optionally pass Ukrainian Clerk strings if needed (out of scope for v1).

## Auto-detect Logic

1. Request arrives at `keyo.studio/`
2. next-intl middleware reads `Accept-Language` header
3. If `uk` or `uk-*` → redirect to `/ua/`
4. Otherwise → redirect to `/en/`
5. After manual locale switch → `NEXT_LOCALE` cookie set → overrides Accept-Language on future visits

## Out of Scope

- Clerk auth UI translations (stays English)
- Dashboard/settings page UI text (added if time permits, otherwise English-only for v1)
- Audio page UI text (same as dashboard — v1 English)
- RTL support (Ukrainian is LTR, not needed)
